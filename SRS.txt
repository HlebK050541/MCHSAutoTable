Требования к проекту
1 Введение
Данный проект создан для того, чтобы оптимизировать действия диспетчеров МЧС по созданию необходимых таблиц. Программное обеспечение называется «Помощник Диспетчеру». Созданное программное обеспечение будет хранить информацию о сотрудниках, о временно нетрудоспособных, ведомствах единой дежурно-диспетчерской службы, а таже данные по проверке связи с данными службами, формировать таблицы в формате word и excel файлах.
2 Требования пользователя
2.1 Программные интерфейсы
Проект будет взаимодействовать с Entity Framework, наборами библиотек по созданию excel и word файлов, такими как: Microsoft.Office.Interop.Excel, Xceed.Document, Xceed.Words.
2.2 Интерфейс пользователя
Система будет взаимодействовать с пользователем по средствам графического интерфейса (Прилагаются мокапы).
2.3 Характеристики пользователей
Данным программным обеспечением сможет пользоваться широкий круг лиц работников МЧС, для того чтобы пользоваться данной программой нужно иметь начальный уровень пользования компьютером.
2.4 Предположения и зависимости
К факторам, которые могут повлиять на требования к системе относиться: хранение данных о пользователях может загружать постоянную память в N-ое количество раз, размер рабочего поля программы.
3 Системные требования

3.1 Функциональные требования

3.1.1 Основные функции
Пользователь имеет возможность использовать приложение без создания аккаунта.

1.	Описание. Пользователь может вносить в БД большое количество данных.
Требование. Приложение должно предоставлять поля для добавления данных в БД приложения. 

2.	Описание. Пользователь может формировать отчеты в word и excel формате.
3.	Требования. Приложение должно обеспечивать вывод информации из БД в файлы форматов word и excel. 

4.	Описание. Пользователь может удалять разные данные из приложения. 
Требование. Приложение должно обеспечивать удаления выбранных данных пользователем. 

3.1.2 Ограничения и исключения

1. Приложение может работать некорректно на неподходящем или слишком слабом аппаратном обеспечении или неподдерживаемых операционных системах.
2. Приложение может некорректно отображать интерфейсы при малом разрешении экрана (800x600 px).

3.2 Нефункциональные требования
3.2.1 АТРИБУТЫ КАЧЕСТВА

3.2.1.1 Требования к удобству использования

1. Доступ к основным функциям приложения не более чем за две операции;
2. Функциональные элементы не перегружают экран;

3.2.1.2 Требования к безопасности

1. Приложение предоставляет возможность добавления, просмотра и удаления записей из БД программы.
2. Для совершения покупок (к примеру дополнительных мест для уведомлений) требуется предпринять дополнительные меры безопасности во избежание потери денежных средств.

3.2.2 Внешние интерфейсы

Окна приложения удобны для использования продолжительное время:

1. функциональные элементы не утомляют глаза.
2. функциональные элементы линейно распределены по экрану.

3.2.3 Ограничения

1. Приложение реализовано на языке программирование С# и при помощи IDE Visual studio 2023;
2. Сведения о работниках, больничных и ЕДДС (Единая дежурно-диспетчерская служба) хранятся в базе данных для последующей вывода на экран.
